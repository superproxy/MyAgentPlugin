# 说明

本 skill 面向 Stitch MCP 的原型生成场景，重点解决三件事：

1. 从自然语言快速生成首版页面
2. 基于用户反馈迭代已有页面
3. 在同一项目内做多方案对比并收敛

## 使用建议

- 首次使用优先创建独立项目，避免污染历史测试项目。
- 提示词先描述信息架构与关键模块，再补视觉风格。
- 每轮只做一类改动（结构或视觉），便于比对效果。

## 最小输入

- 新建生成：页面需求文本
- 迭代编辑：`projectId + 目标 screenId + 修改要求`

## 常见坑

- `projectId` 误传成 `projects/{id}` 全量字符串（应只传 `{id}`）
- `selectedScreenIds` 误带 `screens/` 前缀
- 生成接口超时后立即重试，导致重复任务
