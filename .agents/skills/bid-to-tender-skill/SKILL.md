# 招标文件转投标书 Skill

将招标文件（Word/PDF/文本）自动解析为结构化投标书，覆盖从需求分析到最终交付的全流程。

核心能力：

- 智能解析招标文件，提取关键要求、评分标准、技术规格、商务条款
- 自动生成符合标准投标格式的完整章节结构
- 逐项响应招标要求，生成或引导补充投标内容
- 格式校验确保符合招标方规范（字体、字号、页眉页脚、页码、附件编排）
- 完整性验证与版本控制，确保无遗漏、可追溯

## 适用场景

- 用户上传招标文件，要求"帮我分析这份招标文件"
- 用户要求"根据招标文件生成投标书"
- 用户要求"写技术方案/商务报价书/实施方案"
- 用户要求"检查投标书是否完整/是否符合格式要求"
- 用户要求"对比招标文件和投标书，检查是否有遗漏"
- 用户提供公司资质材料，要求"帮我整理投标资质证明部分"

## 输入模板（建议）

```markdown
项目名称：<投标项目名称>
招标文件：<招标文件内容/路径/链接>
公司信息：
  - 公司全称：
  - 公司简介（核心业务、规模、资质）：
  - 相关案例/业绩：
  - 核心团队：
  - 资质证书清单：
技术能力：
  - 核心技术栈：
  - 技术方案要点（可选，不提供则根据招标要求生成框架）：
  - 项目实施方法论（可选）：
商务信息：
  - 报价策略/预算范围（可选）：
  - 商务条款偏好（可选）：
输出格式：Word / PDF / Markdown
格式要求：<招标方对字体、字号、页边距、页眉页脚等要求，如未提供则按标准投标格式>
补充材料：<公司介绍文档、案例集、资质扫描件等路径或摘要>
```

若输入不完整，先产出首版并标注"待补充项"；不臆造公司资质、业绩、报价等关键信息。

## 执行步骤

### 阶段一：招标文件解析

1. **文档加载与预处理**
   - 识别输入格式（Word/PDF/纯文本/扫描件描述）
   - 提取文档结构（目录、章节、标题层级）
   - 识别表格、列表、加粗/高亮等关键标记

2. **关键信息提取**
   - **项目基本信息**：项目名称、采购编号、采购人、代理机构、预算金额、投标截止时间、开标时间、投标有效期
   - **资质要求**：企业资质、人员资质、业绩要求、财务状况、信用要求
   - **技术规格**：功能需求、性能指标、技术标准、接口要求、兼容性要求
   - **商务条款**：付款方式、交付周期、质保期、售后服务、培训要求、违约责任
   - **评分标准**：价格分、技术分、商务分的权重与评分细则
   - **投标文件格式要求**：封装要求、签字盖章、份数、装订方式、电子版要求
   - **废标条款**：可能导致投标无效的关键条件

3. **响应点识别与分类**
   - 将每个要求标记为：`必须响应` / `加分响应` / `参考信息`
   - 标注响应难度：`直接满足` / `需补充材料` / `需内部确认` / `存在差距`
   - 生成《招标要求-响应对照表》

### 阶段二：投标书结构生成

4. **章节结构规划**
   根据招标文件要求，生成标准投标书目录结构：

   ```markdown
   ## 投标书标准结构

   第一部分：商务文件
   - 1. 投标函
   - 2. 法定代表人授权委托书
   - 3. 投标保证金（如有）
   - 4. 资格证明文件
     - 4.1 营业执照
     - 4.2 资质证书
     - 4.3 财务状况证明
     - 4.4 信用证明
     - 4.5 类似项目业绩
     - 4.6 人员资质证明
   - 5. 商务条款响应/偏离表
   - 6. 中小企业声明函（如适用）

   第二部分：技术文件
   - 7. 技术方案
     - 7.1 项目理解与需求分析
     - 7.2 总体技术架构
     - 7.3 功能实现方案
     - 7.4 关键技术路线
     - 7.5 系统性能保障
     - 7.6 安全方案
   - 8. 实施方案
     - 8.1 项目实施计划
     - 8.2 项目组织与人员配置
     - 8.3 质量管理方案
     - 8.4 风险管理方案
     - 8.5 培训方案
   - 9. 售后服务方案
     - 9.1 质保期服务承诺
     - 9.2 技术支持体系
     - 9.3 应急响应机制
     - 9.4 升级与维护计划
   - 10. 技术条款响应/偏离表

   第三部分：报价文件
   - 11. 开标一览表
   - 12. 分项报价表
   - 13. 价格构成说明

   第四部分：附件
   - 14. 案例证明材料
   - 15. 产品资料/技术白皮书
   - 16. 其他补充材料
   ```

5. **结构适配**
   - 若招标文件有明确的目录要求，严格按招标文件目录生成
   - 若招标文件无明确要求，使用上述标准结构
   - 标注每个章节与招标要求的对应关系

### 阶段三：内容填充

6. **自动生成内容**
   以下内容可根据招标文件自动生成初稿：
   - 投标函（标准模板 + 项目信息填充）
   - 法定代表人授权委托书（标准模板）
   - 商务条款响应/偏离表（逐条对照招标要求）
   - 技术条款响应/偏离表（逐条对照招标要求）
   - 项目理解与需求分析（基于招标文件内容提炼）
   - 开标一览表（结构模板）

7. **引导补充内容**
   以下内容需用户提供信息后生成：
   - 公司概况与核心能力介绍 → 提示用户提供公司介绍材料
   - 类似项目业绩 → 提示用户提供案例列表与证明材料
   - 人员配置 → 提示用户提供核心团队简历
   - 技术方案细节 → 基于招标要求生成框架，提示用户补充技术细节
   - 报价 → 提示用户提供报价策略与分项价格
   - 资质证明 → 提示用户提供资质证书清单

8. **内容生成规则**
   - 技术方案必须逐条响应招标文件中的技术规格要求
   - 每个技术响应点包含：`招标要求原文 → 我方响应方案 → 证明材料索引`
   - 商务条款响应表使用标准格式：`条款号 | 招标要求 | 响应内容 | 响应状态(响应/偏离/不适用) | 说明`
   - 评分标准逐项覆盖，确保得分点不遗漏
   - 所有承诺必须有可验证的支撑材料

### 阶段四：格式校验

9. **格式规范检查**
   根据招标文件要求或标准投标规范，检查以下项目：
   - **页面设置**：纸张大小（A4）、页边距（上下左右）、装订线
   - **字体规范**：标题字体/字号、正文字体/字号、表格字体/字号
   - **段落格式**：行间距、段前段后间距、首行缩进
   - **页眉页脚**：页眉内容（项目名称/公司名称）、页脚（页码格式）
   - **页码**：起始页、编码格式（如"第 X 页 共 Y 页"）
   - **目录**：自动生成目录、目录格式
   - **封面**：项目名称、投标人名称、日期等要素完整性
   - **盖章签字**：盖章位置、签字位置、日期格式
   - **附件编排**：附件编号、附件名称、引用一致性

10. **格式校验输出**
    ```markdown
    ## 格式校验报告

    ### 通过项
    - [x] 页面设置：A4，页边距符合要求
    - [x] 正文字体：宋体小四，符合要求
    ...

    ### 问题项
    | 序号 | 位置 | 问题描述 | 招标要求 | 当前状态 | 修正建议 |
    |---|---|---|---|---|---|
    | 1 | 第三章 技术方案 | 标题字号不符 | 黑体三号 | 黑体小三 | 修改为黑体三号 |
    ...

    ### 缺失项
    - [ ] 缺少"无重大违法记录声明"
    - [ ] 缺少"中小企业声明函"
    ```

### 阶段五：完整性验证

11. **内容完整性检查**
    - 逐条对照招标文件"投标文件组成"要求，检查是否有遗漏章节
    - 检查所有"必须响应"项是否都有对应响应内容
    - 检查评分标准中的每个得分点是否都有覆盖
    - 检查资质证明文件是否齐全
    - 检查签字盖章要求是否满足

12. **一致性检查**
    - 公司名称、项目名称在全文中是否一致
    - 报价金额在不同表格中是否一致（开标一览表 vs 分项报价表）
    - 人员姓名在不同章节是否一致
    - 时间节点是否合理（交付周期 vs 实施计划）
    - 技术参数在不同章节是否一致

13. **版本控制**
    - 每次修改生成新版本号（V1.0 → V1.1 → V2.0）
    - 记录修改日志：`版本号 | 修改日期 | 修改人 | 修改内容摘要 | 修改原因`
    - 保留关键版本节点：初稿、内部评审稿、终稿、盖章版
    - 输出变更对比摘要

## 质量检查清单

- [ ] 招标文件所有"必须响应"项 100% 覆盖
- [ ] 评分标准每个得分点都有对应响应内容
- [ ] 商务偏离表与技术偏离表逐条完整
- [ ] 资质证明材料清单与招标要求一一对应
- [ ] 报价金额在各表格中一致
- [ ] 格式符合招标文件要求（字体/字号/页边距/页眉页脚/页码）
- [ ] 签字盖章位置标注清晰
- [ ] 附件编号与正文引用一致
- [ ] 无敏感信息泄露（如内部成本、未授权人员信息）
- [ ] 版本号与修改日志完整

## 废标条款检查清单（重点）

生成投标书时，必须逐条检查以下常见废标项：

- [ ] 投标函是否按要求签署、盖章
- [ ] 投标有效期是否满足要求
- [ ] 投标保证金是否按要求提交（金额、形式、有效期）
- [ ] 资质证书是否在有效期内
- [ ] 是否满足联合体投标限制（如不允许联合体）
- [ ] 报价是否超过最高限价
- [ ] 交付周期是否满足要求
- [ ] 质保期是否满足最低要求
- [ ] 投标文件份数、装订、密封是否符合要求
- [ ] 电子版格式是否符合要求
- [ ] 是否存在招标文件明确禁止的偏离

## 内容生成原则

1. **忠实响应**：技术方案必须逐条响应招标要求，不遗漏、不跑题
2. **证据支撑**：每个承诺都要有可验证的证明材料（案例、资质、检测报告等）
3. **得分导向**：围绕评分标准组织内容，高分项重点展开，低分项简洁覆盖
4. **风险规避**：废标条款重点标注、重点检查，确保不因格式或程序问题废标
5. **诚实信用**：不虚构资质、不夸大业绩、不虚假承诺；不确定的内容标注"待确认"
6. **专业规范**：语言正式、逻辑清晰、数据准确、排版规范
