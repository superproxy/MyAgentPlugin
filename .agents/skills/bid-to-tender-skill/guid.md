# 招标文件转投标书 Skill 说明

## 功能概述

将招标文件（Word/PDF/文本）自动解析并生成结构化投标书，覆盖从需求分析到最终交付的全流程。

## 核心能力

| 模块 | 功能 | 说明 |
|---|---|---|
| 招标文件解析 | 关键信息提取、响应点识别 | 自动提取资质要求、技术规格、商务条款、评分标准、废标条款 |
| 投标书结构生成 | 标准章节自动生成 | 商务文件 + 技术文件 + 报价文件 + 附件，支持自定义目录 |
| 内容填充 | 自动生成 + 引导补充 | 投标函、偏离表等自动生成；技术方案、报价等引导用户补充 |
| 格式校验 | 字体/字号/页眉页脚/页码检查 | 按招标文件格式要求逐项校验，输出修正建议 |
| 完整性验证 | 章节完整性 + 一致性检查 | 确保无遗漏、无矛盾，废标条款重点检查 |
| 版本控制 | 版本号 + 修改日志 | 支持初稿→评审稿→终稿→盖章版全链路追踪 |

## 使用方式

### 完整流程
```
请根据这份招标文件生成投标书
[上传招标文件 + 提供公司信息]
```

### 仅分析招标文件
```
请分析这份招标文件，重点关注废标条款和评分标准
[上传招标文件]
```

### 仅格式校验
```
请检查这份投标书是否符合招标文件的格式要求
[上传投标书 + 格式要求]
```

### 仅完整性检查
```
请检查投标书是否完整，有没有遗漏
[上传招标文件 + 投标书]
```

## 输入要求

- **必填**：招标文件（Word/PDF/文本）
- **建议提供**：公司信息、资质清单、案例列表、核心团队、报价策略
- **可选**：技术方案要点、商务条款偏好、输出格式偏好

## 输出产物

1. 招标文件分析报告（项目概要、关键要求、废标风险、响应策略）
2. 完整投标书正文（按招标要求或标准结构）
3. 格式校验报告（通过项 + 问题项 + 修正建议）
4. 完整性验证报告（章节完整性 + 响应覆盖率 + 一致性检查 + 待补充项）
5. 版本信息

## 注意事项

- 不虚构公司资质、业绩、报价等关键信息
- 不确定的内容标注"待确认"或"待补充"
- 废标条款重点标注、重点检查
- 所有承诺需有可验证的证明材料
