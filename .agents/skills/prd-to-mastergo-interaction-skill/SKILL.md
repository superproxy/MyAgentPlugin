# PRD -> MasterGo 交互图 Skill

用于把 PRD 快速转成可评审的交互图/交互原型（MasterGo）。

## 输入要求

- 必填：PRD 文本或 PRD 文件内容
- 选填：`shortLink` / `fileId` / `layerId`（用于回写到已有设计文件）

若缺少 `layerId`，优先向用户索取包含 `layer_id` 的 MasterGo 链接。

## 执行步骤

1. 从 PRD 提取三类信息：
   - 页面清单（如：首页、设备连接页、音频控制页）
   - 关键用例（P0 优先）
   - 状态分支（成功、失败、权限拒绝、超时等）
2. 生成交互图骨架：
   - 起点 -> 页面跳转 -> 关键操作 -> 系统反馈 -> 异常回路
   - 每条链路使用“用户动作 -> 系统响应”描述
3. 组织 MasterGo C2D 输入：
   - `data` 中写页面结构、组件区块、交互跳转关系
   - 桌面后台默认宽度 `1440`（移动端则明确标注 iOS/Android）
4. 调用 MasterGo MCP：
   - 先确认 MCP 工具参数 schema
   - 原型生成使用 `mcp__C2d`
5. 输出结果：
   - 交互图覆盖的用例范围
   - 生成结果链接/文件定位信息
   - 未覆盖项与待确认项

## 配色原则（生成时必须应用）

1. **语义色优先**：颜色先服务信息层级与状态表达，再考虑装饰。
2. **主色单一**：全局只设 1 个主色（品牌/主操作），避免多主色竞争注意力。
3. **状态分离**：
   - 成功：绿色系
   - 警告：橙色系
   - 错误：红色系
   - 信息：蓝色系
4. **中性色分层**：背景、边框、正文、次要文字至少 4 级灰阶，保证可读层次。
5. **对比度合规**：
   - 正文与背景对比度建议 >= 4.5:1
   - 大号文字建议 >= 3:1
6. **暗黑模式映射**：同一语义在明暗主题使用等价色，不改变语义。
7. **一致性约束**：同类组件（按钮、标签、告警条）跨页面使用同一色令牌。

## C2D 提示词模板（可直接复用）

```markdown
请根据以下 PRD 生成中保真交互原型，输出页面流程清晰的交互图结构。

【目标】
- 先覆盖 P0 用例，再覆盖 P1
- 每个关键用例包含：进入条件、用户动作、系统反馈、异常分支

【页面清单】
- <页面1>
- <页面2>

【关键用例】
- UC-01: <名称>
- UC-02: <名称>

【交互要求】
- 以“用户动作 -> 系统响应”标注每一步
- 明确成功流与异常流（超时/失败/权限拒绝）
- 页面之间给出跳转关系

【视觉与结构】
- 组件化布局（头部、内容区、操作区、反馈区）
- 保持一致的按钮、状态提示、返回路径
- 应用语义化配色：主色/成功/警告/错误/信息明确区分
- 正文与背景对比度满足可读性要求，避免浅灰字叠浅灰底
- 明暗主题保持同语义同色位映射
```

## 质量门槛

- 至少覆盖全部 P0 用例
- 每个 P0 用例至少 1 条成功流 + 1 条异常流
- 页面跳转无断链（任何页面都可回到主路径）
- 配色满足语义一致性与可读性要求（含明暗主题）
