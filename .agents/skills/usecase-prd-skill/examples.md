# 示例：蓝牙音响/功放控制 App（基于用例）

## 输入摘要（来自企划书）

- 一个 App 统一控制多类型设备（功放、音响）与多芯片方案（杰理、山景）
- 需要设备自动识别并加载对应功能界面
- 核心能力包含：音频控制、音源切换、K 歌、设备灯光与亮度控制
- 需支持深浅色模式、多语言、消息推送与更新提醒

## 4. 详细用例规约
### UC-01 连接并识别设备
- 目标：用户通过蓝牙完成连接，并进入匹配的控制界面
- 参与者：终端用户、App、目标设备
- 前置条件：
  - 手机蓝牙已开启
  - App 已获得蓝牙权限
- 触发条件：用户在首页点击“添加设备”
- 主成功场景（用户操控 -> 系统响应）：
  1. 用户点击“扫描设备”。
  2. 系统展示可发现设备列表（设备名、类型、信号强度）。
  3. 用户选择目标设备并点击“连接”。
  4. 系统建立蓝牙连接并显示连接状态“已连接”。
  5. 系统识别设备类型与芯片方案，自动加载对应控制页面模块。
- 异常/扩展流程：
  - E1 权限拒绝：系统提示授权并引导到系统设置页。
  - E2 连接超时：系统提示重试并保留已扫描列表。
  - E3 设备不兼容：系统提示“不支持的设备型号”并记录诊断码。
- 后置条件：设备进入“可控制”状态，首页展示最近连接设备。
- 验收标准（Given/When/Then）：
  - Given 蓝牙已开启且权限已授权，When 用户点击扫描并连接，Then 5 秒内出现连接结果反馈。
  - Given 连接成功，When 系统识别到设备类型，Then 自动进入对应控制页面且模块与设备能力一致。

### UC-03 切换音源并控制播放
- 目标：用户可在蓝牙/本地音乐/U盘/SD卡/网络电台/FM 间切换并完成播放控制
- 主成功场景（用户操控 -> 系统响应）：
  1. 用户进入“音源”页面选择目标音源。
  2. 系统切换音源并高亮当前来源。
  3. 用户点击“播放/暂停/上一曲/下一曲”。
  4. 系统执行指令并在 500ms 内反馈最新播放状态。
- 异常流程：
  - E1 无可用媒体：系统提示“当前音源无可播放内容”。
  - E2 指令失败：系统提示失败原因并允许重试。
