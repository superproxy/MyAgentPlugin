# usecase-prd-skill 说明

## 这个 skill 做什么

把用户提供的业务材料（企划书、会议纪要、口述需求）转换为基于用例的产品需求文档（PRD），重点表达“用户如何操作功能，系统如何反馈”。

## 什么时候触发

- 用户提到：需求文档、PRD、产品需求、用例分析、用户操作流程、功能拆解
- 用户提供了原始材料，希望整理成可开发可验收文档

## 产出特点

- 有“用例总览 + 详细用例规约”
- 每个关键用例包含：前置、触发、主成功场景、异常分支、后置、验收标准
- 强制使用“用户操控 -> 系统响应”的表达方式

## 配套文件

- `SKILL.md`：规则与模板
- `examples.md`：蓝牙音响项目示例
