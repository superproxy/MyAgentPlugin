# 示例：根据周报生成按模板对齐的月报汇报稿

## 输入

```markdown
报告类型：月报
汇报人：杨学增
时间范围：2026年4月
周报材料：工作周报/###4月###.md
参考模板：工作周报/杨学增-12月报报告.pptx
输出形式：汇报 Markdown
模板模式：严格模板模式
```

## 输出（节选）

```markdown
# 第1页｜封面
- 标题：2026年4月月度报告
- 汇报人：杨学增
- 日期：2026-05-10

# 第2页｜目录
- 01 工作总结
- 02 问题与挑战
- 03 改进措施
- 04 下月计划

# 第3页｜PART 01 工作总结

# 第4页｜工作总结
| 项目 | 主要事项 | 进度 | 总结 |
|---|---|---|---|
| 灵机一动项目 | 电信 web-sdk 上线、宠参谋集成上线、项圈方案评审 | 已上线/已评审 | 完成对外能力落地与新方案推进 |
| IOT 项目 | 奥普华为接入、网关上线、许可证服务开发完成、第三方认证对接 | 已上线/已完成 | 基础平台能力持续补齐 |

# 第5页｜工作总结
| 项目 | 主要事项 | 进度 | 总结 |
|---|---|---|---|
| 灵机 1.9/1.10 | openclaw 对外接口、宠参谋 UI 改版、京东 JoyInside 对接、百度云智能平台对接、宠物运动/档案开发 | 持续开发/部分联调 | 从功能建设转向联调与场景扩展 |
| IOT 平台 | 奥普开放接口提测、蓝牙网关方案与开发、标准版本上线 | 提测/上线 | IOT 版本化与网关能力逐步成型 |

# 第6页｜问题与挑战
- 宠物项圈提测受硬件设备缺失影响，测试推进受限。

- 待确认项：百度云智能平台对接与亚晟新需求是否存在外部依赖风险。

# 第7页｜改进措施
- 对存在硬件依赖的事项提前确认设备与提测窗口，减少等待时间。
- 对联调型需求建立更清晰的里程碑节点，提前锁定接口与责任人。
- 将重复建设能力继续沉淀到标准化版本，降低后续项目接入成本。

# 第8页｜PART 04 下月计划

# 第9页｜下月计划
| 项目 | 计划事项 | 时间/节点 |
|---|---|---|
| 灵机一动项目 | 宠物项圈提测、灵机移动官网上线、台灯项目评审与接口对接、一键创建 agent、工作流开发 | 5 月 |

- 致谢语：谢谢欣赏

```

## 使用提示

- 如果只给周报、不提供 PPT，skill 应退化为普通模式，输出标准结构化汇报稿。
- 如果用户要求“严格按模板”，但模板里存在周报无法支撑的页面内容，需保留页面并写明“待确认项”。
